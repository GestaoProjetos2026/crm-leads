# Audit Module (Step 01 Skeleton)

## Responsibilities
Stores immutable diagnostic findings generated by background workers (e.g. 48h Stagnation). Powers the Director-level analytic reports.

## Current State
- `AuditLog` entity defined.
- Engineered as an **append-only** log:
  - Only `createdAt` timestamp is present. No `updatedAt`.
  - Intended to be enforced with PostgreSQL RLS `FOR INSERT WITH CHECK` rules.
- Tracks `weaknessType` (`Stagnation`, `Low_Conversion`).
- Service and Controller stubs configured and tested.

## Next Steps
- Implement `GET /v1/funnel/audit` to aggregate queries with CTEs for funnel performance.
- Implement Redis caching mechanism for analytic results.
